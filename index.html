TheaPantry is a dynamic mobile application designed to simplify household kitchen inventory management through structured digital tracking. By bridging localized data structures with real-time cloud capabilities, the ecosystem systematically prevents food duplication and visualizes storage inefficiencies. The platform architecture transforms traditional pantry tracking into an intuitive, automated experience tailored for modern households.

SDG 12: Responsible Consumption and Production

TheaPantry systematically minimizes residential food waste by offering clear data analytics and real-time expiration monitoring, driving conscious kitchen consumption patterns.

Lab 1: UI Development

Constructing responsive interface architectures using Declarative Jetpack Compose canvases.

This exercise established the foundation of structural layout systems without relying on legacy XML patterns. Managing row and column alignments built immediate intuition for responsive mobile compositions. The implementation proved critical for rendering multi-device viewports cleanly.

Lab 2: Interaction Handling

Configuring active application state engines to capture and respond to user inputs.

Integrating state management patterns revealed how data properties dynamically manipulate UI states during interactive lifecycles. Handling mutable runtime states allowed components to accurately reflect calculations in real time. This progression moved the interface from static frames to functional structures.

Lab 3: Material Design

Applying unified design systems with Material 3 token mapping and dynamic themes.

Implementing Material 3 components enabled clear visual hierarchies across custom surfaces. The process highlighted how standard token parameters adapt structurally to light and dark theme matrices. It established the core purple aesthetic that defines the system's identity.

Lab 4: Navigation Architecture

Building multi-screen routing pipelines with explicit structured argument passing.

Configuring Compose Navigation controllers simplified screen transitions across complex stack histories. Passing entity indices between screens safely prevented structural tracking errors across dashboard fragments. The model decoupled destination fragments for cleaner class modularity.

Lab 5: Room Local Database

Persisting structural inventory entities locally using SQLite and Room DAOs.

Architecting structured data layers with Room provided persistent storage operations across application launches. Interfacing via explicit Data Access Objects simplified query definitions for stock records. It successfully established the structural database foundation needed for local operations.
